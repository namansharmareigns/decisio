# MoSCoW Prioritization – Decisio

## Must Have
- Create and manage engineering decisions
- Record decision context and assumptions
- Update current project context
- Evaluate decision drift
- Assign risk levels to decisions
- View decision risk dashboard
- Generate explainable risk insights
- Persist decision and context data
## Should Have
- Edit and update existing decisions
- View decision history and audit trail
- Filter decisions by risk level or category
- Mark decisions as reviewed
## Could Have
- Export decision reports
- Notifications for high-risk decisions
- Search functionality for decisions
- Tags or labels for decisions
## Won’t Have 
- Source code analysis or static analysis
- Integration with GitHub, Jira, or CI/CD tools
- Automated decision enforcement
- Role-based authentication and permissions
